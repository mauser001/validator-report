# Validator Report

Website for displaying report data generated by Eth2Overwatch:
https://github.com/mauser001/Eth2Overwatch

MIT - The website and code are free to use, modify, and copy.

## Recent Updates

### Report status section (new)
Each report box now displays a compact status block with support for optional fields sent by Eth2Overwatch:

- BeaconHealthy
- BeaconHealth
- Eth1Syncing
- Eth1CurrentBlock
- Eth1HighestBlock
- Eth1SyncState
- ControllerStatus

Rendered output in each report box:

- Beacon: healthy, unhealthy, or unavailable
- Eth1: synced, syncing, or unavailable
- Sync progress when syncing and both block values are available (current/highest and percent)
- Controller status text with automatic collapse for long messages

Backward compatibility is preserved. Missing status fields are handled gracefully and never break rendering.
Existing validator list rendering is unchanged.

```bash
npm run build
```

## PHP Scripts

- public/php/saveReport.php: Saves report data sent by Eth2Overwatch.
- public/php/loadReport.php: Loads saved report data.
- public/php-data/codes.json: List of valid keys for which report data can be sent.

Example:

```json
{"codeX": true, "codeY": true}
```
